Общие вопросы

Как разместить сайт?
Достаточно перенести файлы, создать и импортировать базу данных и прописать настройки соединения с ней.

Перенос файлов
Файлы основного сайта, размещаются в каталоге /home/demo/public_html.
Файлы дополнительных сайтов следует размещать в каталоге /home/demo/public_html/каталог доп. сайта или /home/demo/public_html/каталог доп.сайта. Как настроить?
Для переноса можно воспользоваться «Диспетчером файлов»

Базы данных
Создать базу данных с помощью «Мастер баз данных MySQL». Имя базы данных будет вида: Ваш логин_имя базы данных, например demo_dbase. При добавлении пользователя в базу данных (шаг3) указать «все права». Запомнить: имя базы данных, имя пользователя и пароль.
С помощью phpMyAdmin импортировать базу данных. Как это сделать?

Настройка
В конфигурационном файле сайта прописать данные для соединения с новой базой данных.
В некоторых случаях может потребоваться настройка версии и модулей PHP.  Настройка PHP


Подробнее об этом здесь http://risp.ru/?page260
Как подключиться по FTP?
Настройка FTP соединения

  • Сервер (Хост): hst3.sibnet.ru

  • Порт: ничего не указывать

  • Учетная запись (Пользователь): demo

  • Пароль: Пароль для входа в панель управления


      • В разделе Учетные записи FTP можно настроить собственные учетные записи для разных групп пользователей, ограничив доступ к сайтам и папкам на сервере. По уже настроенным учетным записям можно скачать конфигурационные файлы настройки файлового менеджера по FTP и SFTP протоколу. По ссылке Команды (новое окно) откроется пошаговая инструкция для настройки.

      Инструкции по настройке файлового менеджера (Total Commander, Filezilla) http://risp.ru/?page260#help_ftp
Как подключиться по SSH/SFTP?
По умолчанию доступ по SSH закрыт для всех пользователей. Для получения доступа следует обратиться в службу технической поддержки.
Какие настройки доступа к базе данных нужно указать на сайте?
Сервер баз данных — localhost.
Имя базы и пользователя базы нужно заранее создать через веб-интерфейс: раздел Базы данных — Мастер баз данных MySQL

Имя базы данных и пользователя будут начинаться с префикса «demo». При заведении пользователя рекомендуем воспользоваться программой «Генератор паролей». При добавлении пользователя в базу данных (шаг 3) рекомендуем поставить отметку «все права». На четвертом шаге будет выведено сообщение об успешном создании базы данных и пользователя с указанием их имен.

Подробнее об этом здесь http://risp.ru/?page264
Как добавить дополнительные сайты?
Панель управления Cpanel настроена таким образом, что при заведении учетной записи, автоматически создаются настройки для основного сайта.

Чтобы добавить дополнительный сайт нужно направить новый домен на dns-сервера нашего хостинга и указать имя домена в разделе Домены – Дополнительные сайты.
При добавлении нового сайта, заполняются поля:

  • Имя нового домена – домен дополнительного сайта

  • Поддомен — при добавлении сайта, автоматически создается поддомен в основном домене учетной записи. Это нужно для того, чтобы можно было размещать сайт и тестировать его работу, если новый домен не зарегистрирован или не направлен на ДНСы нашего хостинга. При добавлении указывается только имя поддомена без имени основного домена.

  • Корневая директория – корневой каталог дополнительного сайта для размещения файлов сайта и установки CMS. Корневой каталог будет размещаться в папке public_html. Путь к корневому каталогу можно впоследствии изменить.

Не забудьте направить новый домен на наши dns-сервера:

  • dns1.risp.ru

  • dns2.risp.ru

Подробнее об этом здесь http://risp.ru/?page262
Как протестировать работу сайта, если домен еще не зарегистрирован (не направлен на хостинг)?
Вы всегда можете воспользоваться доменом третьего уровня вида ваш_логин.rinethost.ru (он заводится автоматически при создании аккаунта), например http://fr0000.rinethost.ru.
Для тестирования работы дополнительного сайта можно использовать домен поддомен_допсайта.ваш_логин.rinethost.ru, например http://test.fr0000.rinethost.ru.
Работа с файлами: копирование, перемещение, изменение прав доступа
Для работы с файлами в CPanel есть удобный Диспетчер файлов, в котором можно скачивать, загружать, удалять, сортировать, изменять права доступа к файлам и папкам.

Для отображения файлов, имя которых начинается с точки (например .htaccess) нужно установить галочку Показать скрытые файлы. Сделать это можно в разделе Параметры диспетчера файлов.

Размещать файлы следует в каталоге public_html.

Функционал диспетчера файлов:

+ файл. Позволяет создать произвольный файл в выбранном каталоге.

+ Папка. Позволяет создать папку.

Копировать. Позволяет скопировать файл или папку задав путь копирование в приведенное поле.

Переместить файл. Позволяет переместить файл с одного каталога в другой при этом указав путь в приведенное поле.

Загрузить. Опция позволяет загружать файлы с локального хранилища на сервер. При этом есть возможность перезаписи существующих файлов.

Скачать на локальный диск.. Позволяет скачать выбранный файл.

Удалить. Можно удалить файл или папку с файлами безвозвратно или в корзину.

Восстановить. Восстановление ранее удаленных файлов при просмотре корзины.

Переименовать. Позволяет изменять имя папок и файлов.

Изменить. Используется примитивный текстовый редактор для изменения файлов.

Редактор исходного кода. Продвинутый редактор который имеет в себе встроенный контроль синтаксиса множества языков программирования.

Редактор HTML. Чем то сродни FrontPage редактору. Позволяет отредактировать код Ваших страничек.

Разрешения. Позволяет изменить права на запись, чтение, изменение файлов и папок.

Просмотреть. Быстрый просмотр содержимого файла.

Извлечь. Распаковать архив.

Сжать. Создать архив выделенных файлов и папок.
